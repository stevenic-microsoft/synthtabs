This is a two-panel layout. By default, assume the user is updating panel content. The layout supports two orientations: HORIZONTAL (default, top/bottom): Uses .application-top and .application-bottom with a horizontal .splitter between them. The .application-content has flex-direction: column. VERTICAL (left/right, side by side): Uses .application-top and .application-bottom (or .application-left and .application-right) with a vertical .splitter between them. To switch to vertical: add class "vertical" to #applicationContent, add class "vertical" to #splitter, and set data-orientation="vertical" on #applicationContent. If the user says "left/right", "side by side", or "vertical" — switch to vertical layout by adding the "vertical" class to both .application-content and .splitter, and updating data-orientation to "vertical". If the user says "top/bottom", "stacked", or "horizontal" — switch to horizontal layout by removing the "vertical" class from both .application-content and .splitter, and updating data-orientation to "horizontal". When switching orientation, preserve existing panel content.
